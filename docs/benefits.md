# claude-peers を使うメリット

## 1. 真の並列開発で時間短縮

単体 Claude は1つのタスクを直列で処理します。claude-peers は複数の Worker が**同時に**別ファイルを編集します。

```
単体 Claude:     Backend → Frontend → テスト = 12分
claude-peers:    Backend ─────┐
                 Frontend ────┤ = 5分
                 テスト ──────┘（Backend完了後）
```

実測で約7分（初回）、改善後は約5分で完了。単体比で **40-60% の時間短縮**。

---

## 2. 双方向コミュニケーション

サブエージェント（Agent ツール）は「投げたら結果を待つだけ」。claude-peers の Worker は作業中に質問できます。

- Worker →「`category_id` の検証ってどこまでやる？」→ Dispatcher
- Dispatcher →「外部キーチェックだけでOK」→ Worker

これにより**仕様の曖昧さを作業中にリアルタイムで解消**できます。

---

## 3. 統合エラーの劇的な削減

サブエージェントで並列開発した場合、各エージェントが**別々の仮定**で書くため統合時にエラーが大量発生します。

| | サブエージェント | claude-peers |
|---|---|---|
| 初回テスト結果 | 70テスト中52失敗 | 32テスト中31成功 |
| 原因 | 型不一致、APIパスずれ、認証方式の違い | — |

claude-peers では Dispatcher が基盤（DB、モデル、認証）を先に作り、**「この `get_current_user` を `Depends` で使え」**と具体的に指示できるため、全 Worker が同じ前提で実装します。

---

## 4. ファイル競合の排他制御

複数エージェントが同じファイルを同時編集すると壊れます。claude-peers は `lock_file` / `unlock_file` で**DB レベルの排他ロック**を提供します。

```
Worker-1: lock_file("main.py") → 編集 → unlock_file("main.py")
Worker-2: lock_file("main.py") → 「Worker-1 がロック中です」 → 別ファイルを先に作業
```

サブエージェントにはこの仕組みがありません。

---

## 5. リアルタイムの進捗可視化

Dispatcher が `list_peers()` と `set_summary` で全 Worker の状況を把握できます。

```
Worker-1（Backend）: TASK-001 完了 — models.py, database.py 作成済み
Worker-2（Frontend）: TASK-002 進行中 — templates/index.html 作業中
Worker-3（Tester）: TASK-003 待機中 — TASK-001 依存
```

サブエージェントは**完了するまで中身が見えません**。

---

## 6. 動的なチーム編成

タスクの規模に応じて Worker を自動起動できます。

- 小規模なゴール → Worker 2人で十分
- 大規模なゴール → Worker 4-5人を起動
- Backend が先に完了 → その Worker を Frontend の補助に再アサイン

Dispatcher が**タスクの性質を見て判断**します。

---

## 7. 軌道修正が可能

サブエージェントは一度投げたら修正できません。claude-peers では:

- 「やっぱり REST じゃなくて GraphQL にして」→ 作業中の Worker に即指示変更
- Worker が問題を発見 → Dispatcher に報告 → 方針を修正して全 Worker に通知

**開発途中での仕様変更に対応可能**です。

---

## 8. セッション永続性

Claude Code が落ちても `--resume` で会話を引き継ぎ。ブローカーの状態は SQLite に保存されているため、**途中から再開可能**です。サブエージェントは揮発的で、落ちたら最初からやり直しです。

---

## 9. 完全ローカル動作

全通信が `127.0.0.1` 内で完結。外部サーバー不要、データが外に出ない。**セキュリティ面でも安心**です。

---
